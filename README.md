## Краткое описание работы

В данной лабораторной работе были изучены принципы архитектуры REST и реализован полноценный Web API на ASP.NET Core с использованием подхода Controller-based API.

**Что было сделано:**

1. Создан проект Web API с поддержкой контроллеров.
2. Настроена генерация документации через Swagger (OpenAPI).
3. Разработаны модели данных (`TaskItem`) и DTO (`CreateTaskDto`, `UpdateTaskDto`) для безопасной передачи данных.
4. Реализован контроллер `TasksController` с полным набором CRUD-операций (GET, POST, PUT, DELETE, PATCH).
5. Добавлены дополнительные методы для поиска, фильтрации, сортировки и получения статистики.
6. Настроена политика CORS для возможности обращения к API с фронтенда.
7. Протестировано API с помощью Swagger UI и расширения REST Client в VS Code.

## **Список реализованных маршрутов**

| HTTP Метод | Маршрут                       | Описание действия                                                  |
| :--------- | :---------------------------- | :----------------------------------------------------------------- |
| `GET`      | `/api/tasks`                  | Получить список всех задач (с опциональной фильтрацией по статусу) |
| `GET`      | `/api/tasks/{id}`             | Получить задачу по идентификатору                                  |
| `POST`     | `/api/tasks`                  | Создать новую задачу                                               |
| `PUT`      | `/api/tasks/{id}`             | Полностью обновить существующую задачу                             |
| `DELETE`   | `/api/tasks/{id}`             | Удалить задачу по идентификатору                                   |
| `PATCH`    | `/api/tasks/{id}/toggle`      | Переключить статус выполнения задачи (выполнена/не выполнена)      |
| `GET`      | `/api/tasks/search`           | Поиск задач по названию (без учета регистра)                       |
| `GET`      | `/api/tasks/priority/{level}` | Фильтрация задач по приоритету (Low/Normal/High)                   |
| `GET`      | `/api/tasks/stats`            | Получение статистики (всего, выполнено, в процессе)                |
| `GET`      | `/api/tasks/sorted`           | Получение отсортированного списка задач                            |

## **Итоговая таблица ASP.NET Core Controller-based API**

| Аспект                    | ASP.NET Core Controllers           |
| :------------------------ | :--------------------------------- |
| **Маршруты**              | `[HttpGet]` атрибут над методом    |
| **Группировка маршрутов** | Класс-контроллер                   |
| **Базовый URL**           | `[Route("api/[controller]")]`      |
| **Параметр пути**         | `(int id)` — параметр метода       |
| **Параметр запроса**      | `[FromQuery] bool? completed`      |
| **Тело запроса**          | `[FromBody] CreateTaskDto dto`     |
| **Ответ 200**             | `return Ok(data)`                  |
| **Ответ 201**             | `return CreatedAtAction(...)`      |
| **Ответ 404**             | `return NotFound(...)`             |
| **Ответ 204**             | `return NoContent()`               |
| **Типизация данных**      | Строгая (C#)                       |
| **Документация**          | Swagger — устанавливается отдельно |

## **1. REST — это архитектурный стиль, а не протокол**

Его принципы (клиент-сервер, stateless, единый интерфейс) универсальны и могут быть реализованы на любом языке программирования и фреймворке.

## **2. Контроллер в ASP.NET Core аналогичен Router в Express.js**

Он предоставляет более строгую типизацию, встроенную валидацию моделей и автоматическую генерацию документации через Swagger.

## **3. Использование DTO критически важно для безопасности данных**

Data Transfer Objects позволяют контролировать, какие поля клиент может отправлять или изменять, предотвращая массовое присваивание (over-posting) служебных полей, таких как `Id` или `CreatedAt`.

## **4. Swagger UI значительно ускоряет разработку и отладку**

Он позволяет тестировать эндпоинты прямо в браузере без необходимости писать дополнительный клиентский код или использовать сторонние приложения.
